I GIT It! Git Messages

Understanding Git Messages

Comprehension of The First GIT Message


The first message provides a status update that informs the user that the branch is two steps ahead of the main branch in the MD file. The user has the ability to stage and commit these changes. It appears as if the file is modified.

Comprehension of The Second GIT Message


In the second returned Git message, a fetch and push is performed. The pull of online files is performed from the URL provided and there is a message which appears saying that there are file conflicts in the diverged branch and main/master copies. The files are modified and a deletion is made. Changes not staged for the commit are performed. The console returns the user to the main/master branch.

Comprehension of The Third GIT Message


The third files provides a list of commits which shows the dates and times aking with the authors of the commits. The message tracks and highlights the history of the commits in the MD file.
